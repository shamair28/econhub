Section 1

Why This Chapter Matters

In Chapter 1, we found the exact point where two lines intersect — a precise, single answer. But many real-world economic problems aren't about a single point. They're about a range of possibilities:

  • What baskets of goods can a consumer afford given their income?
  • What combinations of outputs can a firm produce given limited resources?
  • What allocation satisfies all constraints simultaneously, and which one is best?

These are inequality problems. Instead of finding where lines cross, we find regions — sets of all points that satisfy every constraint at once. And once we have that region, we can optimise over it.

The big idea: An inequality ax + by ≤ c divides the entire plane into two halves. The feasible region of a system of inequalities is where all those halves overlap. Linear programming is the art of finding the best point within that overlap.
Section 2

From Equations to Inequalities

An inequality is like an equation, but instead of "equals," we use one of four comparison signs:

SymbolMeaningExample
<strictly less thanx < 5 means x can be 4.9, 0, −3, but NOT 5
≤less than or equal tox ≤ 5 means x can be 5, 4.9, 0, −3
>strictly greater thanx > 2 means x can be 2.1, 3, 100, but NOT 2
≥greater than or equal tox ≥ 2 means x can be 2, 3, 100

The key difference from an equation: the solution to an inequality is not a single point, but an entire interval or region. For example, x < 5 is satisfied by infinitely many numbers.

One Variable vs. Two Variables

When there is one variable, solutions live on a number line. When there are two variables, solutions live in a plane — a two-dimensional region. This chapter focuses on two-variable inequalities, which are the building blocks of economic feasibility analysis.

✓ Notation reminder: A filled/closed circle (●) on a number line means the endpoint is included (≤ or ≥). An open circle (○) means it is excluded (< or >). On graphs, a solid boundary line means included; a dashed line means excluded.
Section 3

The Three Manipulation Rules

You can rearrange inequalities using algebra — but with one crucial twist compared to equations. Here are the three rules you must know cold:

Rule 1 — Adding or subtracting any number: direction is preserved

If a < b, then a + c < b + c and a − c < b − c for any c.

Example: x − 3 < 2 → add 3 to both sides → x < 5 ✓

Rule 2 — Multiplying or dividing by a positive number: direction is preserved

If a < b and c > 0, then ac < bc.

Example: x/2 ≤ 3 → multiply both sides by 2 → x ≤ 6 ✓

Rule 3 — Multiplying or dividing by a negative number: direction REVERSES

If a < b and c < 0, then ac > bc. The inequality sign flips.

Example: −x < 2 → multiply both sides by −1 → x > −2 ✓

⚠ The most common exam mistake: Forgetting to flip the inequality when multiplying or dividing by a negative. Always ask yourself: "Am I multiplying/dividing by a negative?" If yes — flip the sign.
Worked Example — Solving a Linear Inequality
Solve for x:   3 − 2x ≥ 7
3 − 2x ≥ 7
   −2x ≥ 7 − 3       ← subtract 3 from both sides (Rule 1, direction preserved)
   −2x ≥ 4
     x ≤ −2           ← divide by −2 (Rule 3, direction FLIPS)

Solution: x ≤ −2   — all real numbers less than or equal to −2.

Section 4

Graphing a Single Inequality: Half-Planes

A linear equation in two variables, such as 2x + 3y = 12, describes a line — an infinitely thin set of points. A linear inequality, such as 2x + 3y ≤ 12, describes an entire half-plane: all the points on one side of that line (including the line itself if the inequality is non-strict).

How to Graph a Linear Inequality

  1. Draw the boundary line. Treat the inequality as an equation (2x + 3y = 12) and graph that line. Use a solid line for ≤ or ≥ (boundary included), and a dashed line for < or > (boundary excluded).
  2. Pick a test point not on the line. The origin (0, 0) works if the line doesn't pass through it.
  3. Substitute the test point into the inequality. If it's true, shade the side the test point is on. If it's false, shade the other side.
Worked Example — Graphing 2x + 3y ≤ 12
Graph the inequality 2x + 3y ≤ 12.
Step 1: Draw the boundary line  2x + 3y = 12
        y-intercept: x=0 → y = 4   → point (0, 4)
        x-intercept: y=0 → x = 6   → point (6, 0)
        Draw a SOLID line (≤ includes the boundary).

Step 2: Test point (0, 0):
        2(0) + 3(0) = 0 ≤ 12  ✓  (true)

Step 3: Shade the side containing (0, 0) — below-left of the line.
✓ What if (0,0) is on the line? Pick any other easy point, like (1, 0) or (0, 1), as your test point.
Section 5

Systems of Inequalities and Feasible Regions

Just as we solved systems of equations (finding where two lines cross), we can solve systems of inequalities. But now the "solution" isn't a single point — it's the feasible region: the set of all points satisfying every inequality simultaneously.

Each inequality carves out a half-plane. The feasible region is the intersection of all those half-planes — the area where all conditions are met at once.

Worked Example — Finding a Feasible Region
Find and sketch the feasible region for the system:
x + y ≤ 6
2x + y ≤ 10
x ≥ 0, y ≥ 0 
Boundary lines:
  L1: x + y = 6   →  intercepts (6,0) and (0,6)
  L2: 2x + y = 10 →  intercepts (5,0) and (0,10)

Test point (0,0) in each:
  L1: 0 + 0 = 0 ≤ 6   ✓  → shade below L1
  L2: 0 + 0 = 0 ≤ 10  ✓  → shade below L2
  x ≥ 0 and y ≥ 0       → shade the first quadrant

Feasible region = overlap of all four shaded areas.

Corner points (where boundary lines intersect):
  Origin:       (0, 0)   — intersection of x=0 and y=0
  (5, 0)        — intersection of 2x+y=10 and y=0
  (4, 2)        — intersection of x+y=6 and 2x+y=10  [solve system]
  (0, 6)        — intersection of x+y=6 and x=0

Solving for (4, 2):
  x + y  = 6
  2x + y = 10
  ─────────── subtract Eq.1 from Eq.2:
  x = 4   →   y = 6 − 4 = 2   ✓
Key insight: The feasible region is always a convex polygon (if it exists and is bounded). Its boundary is made up of segments from the constraint lines. The vertices — the corner points — are where pairs of boundary lines meet. Finding these corners always reduces to solving a 2×2 system of equations (Chapter 1 technique).

When Is There No Feasible Region?

It's possible for a system of inequalities to have no feasible region — if the constraints are contradictory (e.g., x ≥ 5 and x ≤ 2 simultaneously). In economics, this would mean the constraints are inconsistent — for instance, a budget that is simultaneously required to be above and below a certain level.

Section 6

Linear Programming and the Corner Point Theorem

Linear programming is the process of maximising or minimising a linear objective function over a feasible region defined by linear inequalities. It is one of the most widely used mathematical tools in economics, operations research, and logistics.

The setup has three components:

  • Decision variables: the quantities you control (e.g., x and y)
  • Constraints: the system of inequalities defining the feasible region
  • Objective function: the linear expression you want to maximise or minimise (e.g., Z = 3x + 2y)

The Corner Point Theorem

Theorem: If a linear objective function has an optimal value (maximum or minimum) over a bounded, non-empty feasible region, then that optimal value is attained at one of the corner points (vertices) of the feasible region.

This is a powerful result. Instead of checking every point in an infinite region, you only need to evaluate the objective function at a small, finite number of corner points and pick the best one.

Procedure

  1. Graph the feasible region from the system of constraints.
  2. Find all corner points by solving each pair of boundary lines as a 2×2 system.
  3. Evaluate the objective function at every corner point.
  4. Select the corner that gives the maximum (or minimum) value.
Worked Example — Maximising a Linear Programme
Maximise Z = 3x + 2y subject to: x + y ≤ 6,  2x + y ≤ 10,  x ≥ 0,  y ≥ 0
Corner points (from Section 5):
  (0, 0),  (5, 0),  (4, 2),  (0, 6)

Evaluate Z = 3x + 2y at each corner:
  (0, 0) → Z = 3(0) + 2(0) = 0
  (5, 0) → Z = 3(5) + 2(0) = 15
  (4, 2) → Z = 3(4) + 2(2) = 12 + 4 = 16  ← maximum
  (0, 6) → Z = 3(0) + 2(6) = 12

Maximum value: Z = 16, achieved at (x, y) = (4, 2).
✓ Intuition for the contour line method: The contour lines of Z = 3x + 2y are parallel lines (all have slope −3/2). As Z increases, the contour line slides outward. The last contour that still touches the feasible region before sliding off is the maximum — and it will always touch a corner.
⚠ Unbounded regions: If the feasible region extends to infinity in the direction of the objective, the maximum (or minimum) may not exist. In practice, economic models usually include non-negativity constraints and resource limits that bound the region.
Section 7

Economics Application: Budget Constraints

The most fundamental inequality in consumer economics is the budget constraint. A consumer with income m, buying goods at prices p₁ and p₂, must satisfy:

p₁x₁ + p₂x₂ ≤ m

Combined with the non-negativity constraints x₁ ≥ 0 and x₂ ≥ 0, this defines a triangular feasible region called the budget set — all the consumption bundles the consumer can afford.

The Budget Line

The boundary of the budget set — where the consumer spends exactly all their income — is the budget line:

p₁x₁ + p₂x₂ = m

In slope-intercept form (solving for x₂):

x₂ = m/p₂ − (p₁/p₂)x₁
FeatureValueInterpretation
x₂-interceptm / p₂Maximum amount of Good 2 if all income spent on it
x₁-interceptm / p₁Maximum amount of Good 1 if all income spent on it
Slope−p₁ / p₂The opportunity cost of Good 1 in terms of Good 2
Worked Example — Setting Up a Budget Constraint
A consumer has income m = 12. Good 1 costs p₁ = 2, Good 2 costs p₂ = 3. Write and interpret the budget constraint.
Budget line:  2x₁ + 3x₂ = 12

x₂-intercept: x₁ = 0 → x₂ = 12/3 = 4   → point (0, 4)
x₁-intercept: x₂ = 0 → x₁ = 12/2 = 6   → point (6, 0)

Slope = −p₁/p₂ = −2/3

Interpretation: for every extra unit of Good 1 bought,
the consumer must give up 2/3 of a unit of Good 2.

Budget set (feasible region): 2x₁ + 3x₂ ≤ 12, x₁ ≥ 0, x₂ ≥ 0

Comparative Statics: How the Budget Line Shifts

ChangeEffect on budget lineGeometric interpretation
Income (m) increasesBoth intercepts increase proportionallyParallel outward shift — consumer can afford more of everything
Income (m) decreasesBoth intercepts decrease proportionallyParallel inward shift
Price p₁ fallsx₁-intercept rises; x₂-intercept unchangedRotation outward around the x₂-intercept
Price p₂ risesx₂-intercept falls; x₁-intercept unchangedRotation inward around the x₁-intercept
Connection to linear programming: A consumer choosing the most preferred affordable bundle is solving a type of optimisation problem over the budget set. When preferences are linear (constant rate of substitution), the optimal bundle is at a corner of the budget set — exactly the corner point theorem. When preferences are curved (the standard case with indifference curves), the optimum is an interior tangency point — but that requires calculus, coming in later chapters.

Does the point (3, 4) satisfy the inequality x + 2y ≤ 10? What about (2, 5)?

Show answer
(3,4): 3 + 2(4) = 11 > 10 ✗ — NOT in feasible region

(2,5): 2 + 2(5) = 12 > 10 ✗ — NOT in feasible region

Problem 3 — Corner points

Find all corner points of the feasible region defined by:
x + y ≤ 8,  x ≤ 5,  x ≥ 0,  y ≥ 0

Show answer 
Corner points:
  (0, 0) — x=0, y=0
  (5, 0) — x=5, y=0
  (5, 3) — x=5 and x+y=8 → y=3
  (0, 8) — x=0 and x+y=8
4 corners: (0,0), (5,0), (5,3), (0,8)
Problem 4 — Linear programming

Using the feasible region from Problem 3, maximise Z = 2x + 5y.

Show answer 
Evaluate Z = 2x + 5y at each corner:
  (0, 0) → Z = 0
  (5, 0) → Z = 10
  (5, 3) → Z = 10 + 15 = 25
  (0, 8) → Z = 40  ← maximum
Maximum Z = 40 at (0, 8)
Problem 5 — Budget constraint

A student has a weekly budget of m = 60. Coffee costs p₁ = 4 per cup; textbook hours at the library (renting a study room) cost p₂ = 6 per hour.

(a) Write the budget constraint and budget line.   (b) Find both intercepts.   (c) What is the slope, and what does it mean economically?

Show answer 
(a) Budget constraint:  4x₁ + 6x₂ ≤ 60
    Budget line:         4x₁ + 6x₂ = 60
    Slope-intercept:     x₂ = 10 − (2/3)x₁

(b) x₁-intercept: x₁ = 60/4 = 15 cups of coffee
    x₂-intercept: x₂ = 60/6 = 10 hours of study room

(c) Slope = −p₁/p₂ = −4/6 = −2/3
    Meaning: each extra cup of coffee costs 2/3 of a
    study-room hour — that is its opportunity cost.
